## 1. 架构设计优化

### 1.1 权限套餐设计优化

**当前设计**: 企业套餐变更时，需要计算权限交集并移除越权权限。

**优化建议**:

1. **权限来源记录**: 在 `user_roles` 和 `role_permissions` 中记录权限来源（套餐/自定义），套餐变更时只调整套餐来源的权限，保留自定义权限
2. **变更预览**: 提供降级预览功能，展示哪些权限会被移除，让管理员确认后再执行
3. **权限快照**: 创建企业时保存套餐权限快照，便于后续对比变更

**理由**: 简化复杂的交集计算，避免误删自定义权限。

### 1.2 多租户数据隔离优化

**当前设计**: 共享数据库 + `tenant_id` 行级隔离。

**优化建议**:

1. **强制 tenant_id 填充**: 代码层面使用 MyBatis-Plus 自动填充，禁止手动不填
2. **租户上下文校验**: 每个企业 API 请求进入后必须有 tenant_id 上下文，不存在直接 401
3. **定时校验任务**: 定期扫描全库检查是否存在数据跨租户，生成报表

**理由**: 虽然设计正确，但实现层面需要防护，避免疏漏。

### 1.3 创建企业初始化流程优化

**当前设计**: 创建企业时自动初始化预设角色 + 默认规则。

**优化建议**:

1. **事务包装**: 整个初始化过程在一个事务中，任意步骤失败回滚全部创建
2. **初始化日志**: 记录每个步骤初始化结果，便于排查问题
3. **重试机制**: 初始化失败支持手动重试，不脏数据

**理由**: 初始化步骤多，必须保证原子性和可观测性。

## 2. 安全性优化

### 2.1 平台-企业边界加固

**当前设计**: 平台 API 使用 `@InterceptorIgnore` 绕过租户隔离，靠 `PlatformAdminContext` 身份校验。

**优化建议**:

1. **强制平台 API 身份校验**: 所有 `/platform/**` API 必须校验平台管理员身份，不存在直接拒绝
2. **租户查询强制 tenantId**: 平台管理员查询企业数据必须通过路径参数 `tenantId`，禁止模糊查询跨租户
3. **权限审计日志**: 记录平台管理员访问企业数据的操作日志，便于审计

### 2.2 RBAC 权限检查优化

**当前设计**: @RequirePerm 注解 AOP 拦截。

**优化建议**:

1. **默认禁止**: 使用 Spring Security 配置，默认所有接口需要认证，公开接口特殊标注
2. **权限缓存失效**: 角色/权限变更后主动删除相关用户权限缓存，不等待过期
3. **超管保护测试**: 编写专门测试用例验证最后一个超管无法被降级/删除

### 2.3 批量导入安全

**当前设计**: 支持 Excel 上传批量导入用户。

**优化建议**:

1. **文件大小限制**: 限制最大 10MB，防止大文件攻击
2. **文件类型校验**: 仅允许 `.xlsx`，校验文件头不是仅靠扩展名
3. **病毒扫描**: 上传文件后进行病毒扫描（可选，生产环境建议）
4. **解析超时控制**: 设置解析超时，防止恶意 XML 炸弹

## 3. 用户体验优化

### 3.1 表单校验增强

**发现问题**: 规格中未明确前端即时校验。

**优化建议**:

1. **前端即时校验**:
   - 必填项为空时，提交按钮禁用 + 提示
   - 手机号格式校验即时提示
   - 密码长度/强度校验即时提示
2. **错误提示定位**: 校验失败时，滚动到错误字段并高亮
3. **表单保存草稿**: 大表单支持自动保存草稿，刷新不丢失

### 3.2 操作体验优化

**优化建议**:

1. **危险操作二次确认**:
   - 删除、停用、禁用操作需要弹窗确认
   - 确认按钮文字描述操作（如"确认删除"），更明确
2. **操作结果消息提示**:
   - 成功/失败都要用 Message 组件提示
   - 成功提示 3 秒自动消失
   - 错误提示保持直到用户关闭
3. **加载状态**:
   - 表格数据加载显示骨架屏
   - 按钮提交后禁用并显示 loading
4. **批量操作**: 支持勾选多条批量处理（如批量启用/停用员工）

### 3.3 表格体验优化

**优化建议**:

1. **默认分页**: 每页 20 条，支持切换 10/20/50/100
2. **记住分页状态**: URL 参数保存 page/pageSize，刷新后保持
3. **关键词搜索**: 支持回车触发搜索，清空按钮快速重置
4. **列宽拖拽**: 允许用户调整列表列宽，LocalStorage 保存

## 4. 性能优化

### 4.1 缓存策略

**优化建议**:

1. **权限缓存**: 用户权限信息缓存 Redis ，过期时间 24 小时，角色变更主动清除
2. **数据看板缓存**: 看板数据缓存 5 分钟，用户手动刷新可更新
3. **商品列表缓存**: 商品列表缓存 1 分钟，变更后主动清除

### 4.2 查询优化

**优化建议**:

1. **分页查询**: 所有列表必须分页，禁止返回全表数据
2. **必要索引**: 给常用搜索字段（手机号、名称、状态、create_time）添加数据库索引
3. **模糊查询**: 模糊查询使用前缀匹配，避免全表扫描

### 4.3 前端构建优化

**优化建议**:

1. **路由懒加载**: 每个模块路由按需加载
2. **第三方库按需引入**: Ant Design 组件按需加载，减少打包体积
3. **静态资源 CDN**: React、Ant Design 等公共包可以使用 CDN 加速

## 5. 测试策略优化

### 5.1 Playwright 自动化测试最佳实践

**项目结构建议**:

```
e2e/
├── playwright.config.ts          # 主配置，设置浏览器、并行度
├── tests/                       # 测试用例按模块分目录
│   ├── auth/
│   │   ├── login.spec.ts        # 登录测试
│   │   └── permission.spec.ts  # 权限越界测试
│   ├── platform-admin/
│   └── enterprise-admin/
├── fixtures/
│   └── test-data.ts            # 测试数据工厂
├── pages/                      # POM 模式封装页面
│   ├── LoginPage.ts
│   └── DashboardPage.ts
└── utils/
    └── auth-helper.ts          # 认证帮助方法
```

**运行配置建议**:

1. **并行执行**: 多工人并行运行，缩短测试时间
2. **失败重试**: 失败用例重试 2 次，排除 flaky test
3. **截图录屏**: 失败用例自动截图和录屏，便于排查
4. **CI 集成**: 每次 PR 提交自动运行验收测试

### 5.2 重点测试场景

**必须重点测试**:

| 场景 | 优先级 | 说明 |
|------|--------|------|
| 权限越界测试 | 高 | 不同角色无法访问超出权限的 API |
| 多租户数据隔离测试 | 高 | 租户 A 无法访问租户 B 数据 |
| 并发扣减测试 | 高 | 积分扣减、库存扣减并发不出现负数 |
| 打卡并发测试 | 高 | 同一用户同一时段并发打卡只有一次成功 |
| 最后超管保护测试 | 高 | 确保至少保留一个超管 |

### 5.3 测试数据管理

**建议**:

1. **测试数据隔离**: 每次测试运行使用独立测试数据库，或测试完清理
2. **数据工厂**: 使用 fixtures 工厂方法生成测试数据，代码复用
3. **敏感数据**: 测试环境不使用生产真实数据，脱敏处理

## 6. 开发流程优化

### 6.1 分期开发建议

**当前** 118 个未完成任务，如果一次性开发周期太长。

**建议分期**:

| 阶段 | 交付内容 | 优先级 |
|------|---------|--------|
| Phase 1 | 平台租户管理 + 企业员工管理 + RBAC + 基础权限 | 高 |
| Phase 2 | 积分规则 + 打卡系统 + 积分账户 | 高 |
| Phase 3 | 虚拟商城 + 订单兑换 + 核销 | 中 |
| Phase 4 | 数据报表 + 通知系统 + 安全增强 | 中 |
| Phase 5 | 前端优化 + 集成测试 + 性能调优 | 低 |

**理由**: 分期交付可以尽早上线核心功能，快速获得反馈。

### 6.2 代码评审检查清单

建议代码评审重点检查:

1. **权限检查**: 是否每个 API 都添加了正确的 `@RequirePerm`
2. **租户隔离**: 是否不小心漏掉了 `tenant_id`
3. **错误处理**: 参数校验是否完整，异常是否正确抛 `BizException`
4. **事务边界**: 多步操作是否正确添加事务
5. **并发安全**: 扣减操作是否使用乐观锁

### 6.3 文档维护

**建议**:

1. **API 文档**: 使用 Swagger/OpenAPI 自动生成 API 文档
2. **变更日志**: 每个版本更新记录变更内容
3. **部署文档**: 清楚说明环境要求、配置项、启动步骤
