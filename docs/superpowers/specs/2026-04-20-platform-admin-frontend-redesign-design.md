# 平台后台管理前端重新设计方案

> 日期: 2026-04-20
> 状态: Approved
> 范围: 平台后台四个页面的 UI/交互重新设计，不涉及后端 API 变更
> 设计文档: `docs/superpowers/specs/2026-04-19-multi-product-saas-platform-design.md` Section 6

## 1. 背景

当前平台后台前端（platform-frontend）的产品管理、积木组件库、套餐管理三个页面与多产品 SaaS 平台设计文档中的要求存在偏差，企业管理中套餐驱动的权限展示不够清晰。本方案对四个页面进行定向重新设计。

## 2. 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 产品管理定位 | 保留向导，改为"产品配置向导" | 产品模块是代码注册的，UI 只能配置已有模块的实例 |
| 套餐功能点展示 | 树形折叠面板 | 清晰展示套餐→产品→功能点三层嵌套关系 |
| 企业权限展示 | 权限链路图 + 折叠面板 | 链路可视化直观展示权限推导过程，折叠面板提供细节 |
| 积木组件库 | 保持只读 + 增加扩展引导 | 积木是技术能力清单，不需要 CRUD 管理 |

## 3. 产品管理 — 重新定位

### 3.1 变更内容

**向导重命名**："新建产品向导" → "产品配置向导"

**新增提示**：向导首页（Step 0: 基本信息）顶部加 Alert：
> 本向导基于已注册的产品模块配置产品实例。如需全新产品类型，请联系开发团队扩展积木组件库。

**文案调整**：
- 按钮文案 "新建产品向导" → "配置产品"
- Steps 标题保持不变（基本信息 → 选择触发器 → 组装规则链 → 选择功能点）
- Step 1 选择触发器时，只展示 ProductRegistry 已注册的模块，底部加扩展引导 Alert
- Step 2 组装规则链时，"可用规则节点"列表底部加扩展引导 Alert
- Step 3 选择功能点时，功能点列表底部加扩展引导 Alert

**产品列表新增列**：
- 新增"关联套餐"列：展示引用该产品的套餐名称列表（Tag 形式），无关联时显示"—"

**保留不变**：
- 快速创建/编辑 Modal
- 产品详情 Drawer（规则链预览 + 功能点列表）
- 功能点配置 Modal

### 3.2 扩展引导 Alert 统一格式

所有扩展引导使用 Ant Design `<Alert type="info">` 统一格式：

```
title: "需要更多组件？"
description: "如需新的触发器/规则节点/功能点类型，请联系开发团队扩展积木组件库。"
```

在以下位置出现：
1. 产品配置向导 — Step 1（选择触发器）底部
2. 产品配置向导 — Step 2（组装规则链）"可用规则节点"区域底部
3. 产品配置向导 — Step 3（选择功能点）底部
4. 积木组件库 — 每个 Tab 底部

## 4. 套餐管理 — 树形折叠面板

### 4.1 变更内容

**"配置产品"弹窗重构**：从平铺列表改为树形折叠面板。

**Collapse Panel 结构**（每个产品一个 Panel）：
- **Panel Header**：产品名称 + 触发类型 Tag（蓝=打卡/绿=计步）+ "X/Y 功能点已启用" 统计
- **Panel Body**：功能点列表，每行包含：
  - 必需/可选标签（必需=红, 可选=灰）
  - 功能点名称
  - Switch 开关（必需功能点 disabled，始终开启）
  - 配置值输入框（仅 config 类型功能点，且已启用时显示）
- **Panel Body 顶部**：Alert 提示 "必需功能点不可关闭，可选功能点可自由开关"
- **Panel Body 底部**："保存该产品功能点" 按钮

**保留不变**：
- 套餐列表页（表格 + 筛选）
- 创建/编辑套餐 Modal
- 套餐状态管理（启用/禁用）
- 删除套餐逻辑

### 4.2 交互细节

- 默认展开所有已选产品的 Panel，未选产品的 Panel 不显示
- Checkbox 勾选产品后自动展开该产品的 Panel
- 取消勾选产品后收起并移除 Panel
- 保存按钮按产品独立操作（保存单个产品的功能点配置）

## 5. 企业管理 — 权限链路图

### 5.1 变更内容

**新增 Tab**：企业详情 Modal 增加 "权限总览" Tab，与 "基本信息"、"用户管理"、"已启用产品" 并列。

**权限总览 Tab 结构**：

#### 第一区：链路可视化

横向四段式链路图：

```
[套餐: 标准版] → [包含产品: 2个] → [启用功能点: 8个] → [企业菜单: 6个]
```

每段用不同颜色的圆角卡片展示：
- 套餐段：蓝色（#ddf4ff）
- 产品段：绿色（#dafbe1）
- 功能点段：黄色（#fff8c5）
- 企业菜单段：紫色（#f1e8ff）

段之间用箭头连接。

#### 第二区：产品权限明细

折叠面板，每个产品一个 Panel：

- **Panel Header**：产品名称 + 触发类型 Tag + "X/Y 已启用"
- **Panel Body**：表格，每行一个功能点，列包含：
  - 类型（必需/可选）
  - 功能点名称
  - 启用状态（● 启用 / ○ 禁用）
  - 对应的企业菜单（如"打卡管理 / 时段规则配置"）
  - 未启用功能点的企业菜单列显示灰色 "— 未启用，无菜单"

#### 数据来源

- 套餐信息：来自企业绑定的 `package_id`，查询套餐包含的产品和功能点
- 企业菜单映射：功能点类型 → 企业管理后台菜单的映射关系，在前端维护一个静态映射表（功能点 ID → 菜单路径）

### 5.2 保留不变

- 企业列表页
- 开通企业 Modal
- 基本信息Tab（套餐管理、状态等）
- 用户管理 Tab
- 已启用产品 Tab（可作为简化视图保留）

## 6. 积木组件库 — 扩展引导

### 6.1 变更内容

**新增扩展引导**：每个 Tab（触发器/规则节点/功能点模板）的表格下方加扩展引导 Alert（格式见 3.2）。

**Empty 状态优化**：当某个 Tab 数据为空时，Empty 提示文案改为：

```
"暂无已注册的{触发器/规则节点/功能点}。请联系开发团队添加新的组件类型。"
```

**保留不变**：
- 三个 Tab 结构（触发器/规则节点/功能点模板）
- 已注册产品模块概览区域
- 所有表格列和渲染逻辑

## 7. 前端映射表：功能点 → 企业菜单

企业权限总览中需要将功能点映射到企业管理后台的菜单路径。这个映射关系在前端维护：

```typescript
const FEATURE_MENU_MAP: Record<string, string[]> = {
  // 爬楼积分产品功能点
  time_slot: ['爬楼积分管理', '时段规则配置'],
  special_date: ['爬楼积分管理', '节假日翻倍配置'],
  weekly_gift: ['爬楼积分管理', '周三活动配置'],
  consecutive_reward: ['爬楼积分管理', '连续打卡奖励配置'],
  points_exchange: ['积分商城'],
  daily_cap: ['爬楼积分管理', '每日上限配置'],

  // 走路积分产品功能点
  step_calc_config: ['走路积分管理', '步数换算配置'],
  fun_equivalence: ['走路积分管理', '趣味等价物配置'],
};
```

映射逻辑：
- `points_exchange`（积分兑换）是跨产品共享功能，映射到共享菜单 "积分商城"
- 每个功能点可映射到多层菜单路径（数组表示层级）
- 未在映射表中的功能点显示为 "— 菜单映射未定义"

## 8. 不涉及变更的部分

- 后端 API — 所有数据均来自现有 API，无新增端点
- 数据库 — 无表结构变更
- 路由结构 — 页面路由不变，仅修改页面内部组件
- 菜单结构 — App.tsx 中的侧边栏菜单不变
- 企业前端 / H5 前端 — 不在本方案范围内

## 9. 实现优先级

建议实现顺序：

1. **积木组件库扩展引导**（最简单，纯 Alert 添加）
2. **产品管理重定位**（文案调整 + Alert + 新增列）
3. **套餐管理树形折叠面板**（弹窗重构）
4. **企业管理权限链路图**（新增 Tab + 链路可视化 + 折叠面板）
