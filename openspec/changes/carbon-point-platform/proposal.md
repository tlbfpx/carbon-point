## Why

企业缺乏低成本、高参与度的员工健康管理工具。爬楼梯打卡送积分系统通过游戏化机制激励员工运动，积分兑换虚拟奖品增强参与感。平台以多租户 SaaS 形式交付，注册企业可独立配置规则、管理商品、运营积分体系，无需独立部署。

## What Changes

- 新建多租户 SaaS 平台，支持企业注册、开通、停用的全生命周期管理
- 新建用户端 H5 应用，嵌入微信小程序 WebView 和已有 APP WebView，提供打卡、积分查看、商城兑换功能
- 新建企业管理后台，提供员工管理、积分规则引擎配置、虚拟商品管理、订单管理、积分运营、角色权限管理
- 新建平台运营后台，提供企业管理、全平台数据看板、系统管理
- 打卡积分采用时段随机机制：企业配置多个时段（如早中晚），每个时段设定积分范围，用户打卡时随机获得积分
- 积分规则引擎支持：时段规则、连续打卡奖励、特殊日期翻倍、用户等级系数、每日积分上限，时段不可重叠
- RBAC 权限体系，菜单+按钮+API 三级控制，每企业独立，预设角色+自定义角色
- 用户通过企业邀请链接或管理员批量导入两种方式加入企业，一人仅属一企业
- 商品以虚拟商品为主（优惠券码、直充、权益激活），支持核销管理

## Capabilities

### New Capabilities

- `multi-tenant`: 多租户基础设施——租户（企业）注册、开通、停用、套餐管理，数据逻辑隔离（tenant_id），租户级配置隔离
- `user-management`: 用户管理——注册/登录（JWT）、企业邀请链接加入、管理员批量导入、用户启停、用户与企业一对一绑定
- `check-in`: 打卡系统——按时段打卡、随机积分计算、打卡记录、时段规则校验、防重复打卡
- `point-engine`: 积分规则引擎——时段规则、连续打卡奖励、特殊日期翻倍、用户等级系数、每日积分上限、积分四舍五入、规则防重叠校验
- `point-account`: 积分账户——积分余额管理、积分流水、手动发放/扣减、积分统计
- `virtual-mall`: 虚拟积分商城——虚拟商品管理（优惠券码/直充/权益）、上下架、库存、积分定价、兑换下单、卡券核销
- `rbac`: RBAC 权限控制——角色管理、权限定义（菜单/按钮/API）、用户-角色关联、预设角色模板、自定义角色、至少保留一超管约束
- `reporting`: 数据报表——企业级数据看板（积分趋势、活跃人数、热门商品）、平台级数据看板（企业数/总用户/积分总量/兑换总量）、数据导出
- `h5-user-app`: 用户端 H5 应用——首页概览、打卡页面（时段展示/一键打卡/结果动画）、积分页面（余额/明细/排行榜/等级）、商城页面（商品列表/兑换/卡券包）
- `enterprise-admin`: 企业管理后台——员工管理、规则配置、商品管理、订单管理、积分运营、角色权限管理、数据看板
- `platform-admin`: 平台运营后台——企业管理（列表/开通/停用）、全平台数据看板、平台配置、系统管理（管理员/操作日志/权限）

### Modified Capabilities

（无，全新项目）

## Impact

- **前端**：React + Ant Design + Vite，pnpm Monorepo（apps/h5 + apps/dashboard，packages 共享组件/API 层/hooks/工具）
- **后端**：Spring Boot 3.x + Java 21，模块化（carbon-common / carbon-system / carbon-checkin / carbon-points / carbon-mall / carbon-report / carbon-app）
- **ORM**：MyBatis-Plus，TenantLineInnerInterceptor 实现多租户自动隔离
- **数据层**：MySQL（主库）+ Redis（会话/权限缓存）+ OSS（文件存储）
- **API**：RESTful API，JWT 认证，统一响应封装，全局异常处理
- **部署**：前后端分离，H5 独立部署，管理后台独立部署，后端单应用部署
