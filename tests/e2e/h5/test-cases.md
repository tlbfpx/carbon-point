# H5端核心功能测试用例

> 测试环境：`http://localhost:3002/h5`
> 技术栈：React + Ant Design Mobile + React Router + React Query
> 路由前缀：`/h5/`
> 认证方式：JWT（access_token + refresh_token，持久化于 localStorage `carbon-auth`）

---

## 1. 登录功能（LoginPage）

### 1.1 正常登录

- **Given**：用户已注册（手机号: `13900000004`，密码: `Test123456!`）
- **When**：用户在登录页输入正确手机号和密码，点击"登录"
- **Then**：登录成功，显示 Toast"登录成功"，跳转到首页 `/`，显示用户积分和打卡状态

### 1.2 空字段提交

- **Given**：用户在登录页
- **When**：用户未输入手机号和密码，直接点击"登录"
- **Then**：显示 Toast"请输入手机号和密码"，不发起 API 请求

### 1.3 手机号格式错误

- **Given**：用户在登录页
- **When**：用户输入格式错误的手机号（如 `1390000` 或 `abcdefghijk`）
- **Then**：前端校验失败或 API 返回格式错误提示，不跳转页面

### 1.4 密码错误

- **Given**：用户已注册（手机号: `13900000004`，密码: `Test123456!`）
- **When**：用户输入正确手机号但错误密码 `wrongpassword`
- **Then**：API 返回错误，显示 Toast"登录失败"或具体错误信息，停留在登录页

### 1.5 未注册手机号

- **Given**：数据库中不存在手机号 `13800000001`
- **When**：用户输入未注册手机号 `13800000001` 和任意密码
- **Then**：API 返回错误，显示 Toast"登录失败"或"用户不存在"，停留在登录页

### 1.6 记住我勾选

- **Given**：用户在登录页
- **When**：用户勾选"记住我"，输入正确凭证后登录
- **Then**：登录成功，`remember` 字段随请求发送给后端，Token 被持久化存储

### 1.7 登录后跳转到注册页

- **Given**：用户已登录（Token 有效）
- **When**：用户直接访问 `/h5/login`
- **Then**：路由守卫检测到已认证，自动重定向到首页 `/`

### 1.8 网络错误

- **Given**：用户输入正确凭证
- **When**：网络断开或 API 超时（30s）
- **Then**：显示 Toast"网络错误，请重试"，停留在登录页

### 1.9 登录后数据刷新

- **Given**：用户登录成功，跳转到首页
- **When**：首页加载时
- **Then**：`checkInStatus`、`leaderboardHistory`、`leaderboardContext` 等 Query 同时发起请求，显示用户打卡状态、连续天数和排行榜

## 2. 注册功能（RegisterPage）

### 2.1 正常注册

- **Given**：用户未注册
- **When**：用户输入手机号 `13900000099`、6位短信验证码 `123456`、密码 `Test123456!`、确认密码 `Test123456!`，点击"注册"
- **Then**：注册成功，显示 Toast"注册成功"，跳转到登录页 `/login`

### 2.2 发送验证码 - 正常

- **Given**：用户未注册
- **When**：用户输入正确的11位手机号 `13900000099`，点击"获取验证码"
- **Then**：发送短信验证码请求，显示 Toast"验证码已发送"，按钮变为倒计时 60s

### 2.3 发送验证码 - 手机号格式错误

- **Given**：用户在注册页
- **When**：用户输入错误手机号 `1390000`，点击"获取验证码"
- **Then**：显示 Toast"请输入正确的手机号"，不发送请求

### 2.4 发送验证码 - 倒计时中

- **Given**：用户刚点击"获取验证码"，倒计时显示 `59s`
- **When**：用户再次点击"获取验证码"
- **Then**：按钮保持禁用状态，不重复发送请求

### 2.5 密码不一致

- **Given**：用户在注册页
- **When**：用户输入密码 `Test123456!` 和确认密码 `Test1234567`
- **Then**：显示 Toast"两次密码不一致"，不发起注册请求

### 2.6 密码长度不足

- **Given**：用户在注册页
- **When**：用户输入密码 `Test12`（少于8位）
- **Then**：显示 Toast"密码长度至少8位"，不发起注册请求

### 2.7 字段为空

- **Given**：用户在注册页
- **When**：用户未填写手机号、验证码或密码，直接点击"注册"
- **Then**：显示 Toast"请填写完整信息"，不发起请求

### 2.8 邀请码（选填）

- **Given**：用户有有效邀请码 `INVITE2026`
- **When**：用户填写完整信息，并在邀请码字段输入 `INVITE2026`
- **Then**：注册请求中包含 `inviteCode` 字段，注册成功

### 2.9 注册后 Token 状态

- **Given**：用户注册成功后跳转到登录页
- **When**：用户在登录页输入刚注册的凭证登录
- **Then**：登录成功，Token 正确存储，用户状态正确

## 4. 打卡功能（CheckInPage）

### 4.1 正常打卡（可用时段）

- **Given**：用户已登录，存在状态为 `available` 的打卡时段
- **When**：用户点击该时段的"打卡"按钮
- **Then**：调用 `POST /checkin`，成功后进入打卡成功动画，显示绿色勾选 + 获得积分数，3秒后自动返回首页；`checkInStatus`、`pointsAccount`、`timeSlots` Query 被 invalidate

### 4.2 已打卡时段不可重复打卡

- **Given**：用户已登录，某时段状态为 `checked_in`
- **When**：用户看到该时段卡片
- **Then**：该时段显示"已完成"按钮（禁用状态），背景色为 `#f6ffed`（浅绿）

### 4.3 未开始时段

- **Given**：用户已登录，某时段状态为 `not_started`
- **When**：用户看到该时段卡片
- **Then**：该时段显示"暂不可打卡"按钮（禁用状态），背景色为灰色

### 4.4 已结束时段

- **Given**：用户已登录，某时段状态为 `ended`
- **When**：用户看到该时段卡片
- **Then**：该时段显示"暂不可打卡"按钮（禁用状态），背景色为灰色

### 4.5 无可用时段

- **Given**：用户已登录，但暂无可用打卡时段
- **When**：用户访问打卡页
- **Then**：显示"暂无可用打卡时段"

### 4.6 打卡成功动画

- **Given**：用户刚完成打卡，API 返回成功
- **When**：打卡成功页面展示
- **Then**：显示绿色圆圈 ✓ 动画 + 浮动的 `+X` 积分数字动画、3秒倒计时提示、跳过按钮

### 4.7 打卡 API 失败

- **Given**：用户点击"打卡"按钮
- **When**：打卡 API 返回错误（如并发冲突）
- **Then**：显示 Toast"打卡失败"或具体错误信息，不跳转

### 4.8 打卡网络错误

- **Given**：用户点击"打卡"按钮
- **When**：网络断开
- **Then**：显示 Toast"网络错误，请重试"

### 4.9 查看打卡历史

- **Given**：用户在打卡页
- **When**：用户点击"查看打卡历史 →"
- **Then**：跳转到 `/checkin/history`

### 4.10 打卡时段显示格式

- **Given**：后端返回时段 `startTime: "08:00:00", endTime: "10:00:00"`
- **When**：打卡页渲染时段卡片
- **Then**：显示为 `08:00 - 10:00`（去掉了秒）

### 4.11 打卡后立即返回首页

- **Given**：用户完成打卡，在打卡成功页面
- **When**：用户点击"立即返回"按钮
- **Then**：立即跳转到首页，不等待倒计时

### 4.12 打卡时段 badge 颜色

- **Given**：用户已登录，各时段状态正确
- **When**：打卡页渲染时段列表
- **Then**：已打卡 → 绿色"已打卡"，可打卡 → 蓝色"可打卡"，未开始 → 灰色"未开始"，已结束 → 灰色"已结束"

## 6. 积分查询（PointsPage）

### 6.1 积分页加载

- **Given**：用户已登录，有积分数据
- **When**：用户访问 `/points`
- **Then**：顶部显示总积分数、可用积分、等级（Lv.X 名称）、距离下一等级的积分差距

### 6.2 等级进度条

- **Given**：用户已登录，总积分 `3500`
- **When**：用户在积分页
- **Then**：进度条显示从 Lv.1 青铜（0-999）到 Lv.2 白银（1000-4999）的进度，进度 = `(3500-1000)/(5000-1000)*100 = 62.5%`，显示 63%

### 6.3 各等级配置

- **Given**：用户已登录
- **When**：用户在积分页查看等级
- **Then**：Lv.1 青铜(0-999)、Lv.2 白银(1000-4999)、Lv.3 黄金(5000-19999)、Lv.4 铂金(20000-49999)、Lv.5 钻石(50000+)，进度条颜色对应等级

### 6.4 积分明细显示

- **Given**：用户已登录，有积分变动记录
- **When**：用户在积分页
- **Then**：显示最近5条积分明细，每条显示描述和变动数（正数绿色 `+`，负数红色 `-`）

### 6.5 积分明细为空

- **Given**：用户已登录，但暂无积分明细
- **When**：用户在积分页
- **Then**：积分明细列表为空

### 6.6 排行榜显示

- **Given**：用户已登录，有排行榜数据
- **When**：用户在积分页
- **Then**：显示完整排行榜，包含我的排名卡片（超越 X% 用户）和各排名项（带金银铜颜色标记）

### 6.7 我的排名显示

- **Given**：用户已登录，在排行榜中有排名
- **When**：用户在积分页查看排行榜
- **Then**：排行榜上方显示蓝色"我的排名"卡片，显示排名号和超越百分比

### 6.8 积分页 TabBar

- **Given**：用户在积分页
- **When**：用户点击 TabBar 中的"首页"标签
- **Then**：跳转到 `/`

### 6.9 积分页无数据

- **Given**：用户已登录，但积分账户数据为空
- **When**：用户在积分页
- **Then**：总积分显示 `0`，等级为 Lv.1 青铜，进度条为空

## 13. 认证与路由守卫

### 13.1 未登录访问受保护路由

- **Given**：用户未登录（localStorage `carbon-auth` 中无 Token）
- **When**：用户尝试访问 `/` 或 `/checkin` 等受保护路由
- **Then**：路由守卫检测到 `isAuthenticated === false`，重定向到 `/login`，完成后 URL 为 `/login`

### 13.2 已登录访问登录页

- **Given**：用户已登录（Token 有效）
- **When**：用户访问 `/login`
- **Then**：路由守卫检测到已认证，`Navigate to="/"` 替换当前历史记录

### 13.3 已登录访问注册页

- **Given**：用户已登录（Token 有效）
- **When**：用户访问 `/register`
- **Then**：路由守卫不拦截（注册页为公开路由），页面正常显示

### 13.4 Token 过期自动刷新

- **Given**：用户已登录，access_token 过期，refresh_token 有效
- **When**：用户发起 API 请求（如访问首页数据）
- **Then**：API 返回 401，axios 拦截器自动调用 `/auth/refresh`，获取新 Token 后重试原请求，用户无感知

### 13.5 Token 全部过期

- **Given**：用户已登录，access_token 和 refresh_token 均过期/无效
- **When**：用户发起 API 请求
- **Then**：refresh 失败，调用 `logout()` 清空状态，重定向到 `/login`

### 13.6 页面刷新后登录状态保持

- **Given**：用户已登录，`carbon-auth` 数据在 localStorage 中
- **When**：用户刷新页面（F5）
- **Then**：Zustand `persist` 中间件从 localStorage 恢复 Token 和用户状态，页面无需重新登录

### 15.1 完整打卡流程

- **Given**：用户已登录，今日未打卡，有可用时段
- **When**：用户进入首页 → 点击"立即打卡" → 在打卡页点击某时段的"打卡"按钮 → 等待动画 → 返回首页

### 15.2 完整兑换流程

- **Given**：用户已登录，积分充足，商品有库存
- **When**：用户进入商城 → 点击商品 → 查看详情 → 点击"立即兑换" → 确认 Dialog → 等待结果
- **Then**：兑换成功，Toast"兑换成功"，跳转到我的卡券页，积分减少，卡券列表刷新

- **When**：用户进入商品详情 → 点击"立即兑换" → 弹出 Dialog → 输入手机号"13900000001" → 点击"确认"
- **Then**：兑换请求包含 `extraInfo: { account: "13900000001" }`，兑换成功，跳转到卡券页

### 15.4 积分不足兑换失败

- **Given**：用户已登录，但积分不足
- **When**：用户尝试兑换高积分商品

### 15.5 打卡后查看积分明细

- **When**：用户进入积分页查看明细

### 15.7 TabBar 完整导航路径
