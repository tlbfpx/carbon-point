# 5 人开发团队配置

## 团队组成

| 角色 | 职责 | 负责模块 |
|------|------|---------|
| **架构师/ Tech Lead** (1人) | 技术决策、架构设计、代码审查、关键模块开发 | 全模块技术指导 |
| **后端开发工程师** (2人) | 后端 API 开发、数据库设计、业务逻辑实现 | 后端各模块 |
| **前端开发工程师** (1人) | 前端页面开发、组件设计、用户交互实现 | 企业端/平台端/H5 |
| **测试/ DevOps 工程师** (1人) | 测试用例、CI/CD、部署运维、监控告警 | 质量保障+基础设施 |

## 协作流程

### Git 分支策略
```
main
  └── develop
        ├── feature/backend-1-unified-resources
        ├── feature/backend-2-business-modules
        ├── feature/frontend-resource-management
        └── feature/test-automation
```

### 代码审查流程
1. 开发者创建 feature 分支
2. 提交 Pull Request
3. 至少 1 人审查（架构师或另一个后端）
4. CI 测试通过
5. 合并到 develop 分支

### 每日站会
- 时间：每天上午 10:00
- 时长：15-30 分钟
- 内容：昨天进度、今天计划、遇到的问题

### 周例会
- 时间：每周五下午 14:00-16:00
- 内容：本周总结、下周计划、技术分享
